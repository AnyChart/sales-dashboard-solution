    Sales Dashboard "Wine Sales in France"

    This is an example which covers the basic sales dashboard items in our opinion. We have tried to make it a comfortable, visually appealing, and easy to grasp.
    Example contains four tabs: The first tab - is a general view - which allows to assess the situation as a whole, the other three tabs intend to reveal the situations of particular categories of products, sales team advances and evaluation of sales by region.
    The solution is dedicated to the wines of France, because we believe that this topic is pleasant thing to think about.

    Replacing Data

    All data, used in this sample are collected in src/data/data.js file.
    The structure of the data is (all 4 data sets have same first level: "YTD", "QTD", "MTD", "WTD" and "all". And the structure for this five is the same, so this level will be skiped further):

    generatedGeneralData - data for "General" tab.
    • revenue_chart - table with rows like ["2001", 354165, 457]
    • key_metrics - has several objects and the data for each of them.
    • five_best
      • sales_men - table with ["Katherine Dean", 505625] rows
      • regions - table with ["Katherine Dean", 505625] rows
      • products - table with ["Katherine Dean", 505625] rows
    generatedProductsData - data for "Sales by Product" tab.
    • categories_data - data of thr products by categories. The biggest data set in the sample.
      • x - category name
      • value - value for the whole category
      • data - data for the table and map chart on this tab
    generatedRegionsData - data for "Sales by Region" tab.
    • regions_data - list of all regions with data for each of them, such as:
      • id - id of region
      • x - name of region
      • value - value for region
      • market_share - the share of our products in the region
      • total_share - the share of sales in this region from the total sales
      • revenue - table data for revenue chart [["2001", 480973, 490]
    generatedSalesTeamData - data for "Sales Team" tab.
    • team_data - list of all person with data for each of them
